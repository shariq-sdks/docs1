## Authentication

All API endpoints require authentication using your service credentials. See our [Authentication Guide](/guides/authentication) for details.

## API Categories

### Service Management
Manage your authorization server configuration, including service settings, supported grant types, and security policies.

### Client Management
Register and manage OAuth 2.0 clients, including client credentials, redirect URIs, and scopes.

### Authorization Flow
Handle OAuth 2.0 authorization requests, including user consent and authorization code generation.

### Token Management
Issue, validate, and revoke access tokens and refresh tokens.

### User Management
Manage user accounts, authentication, and profile information.

### Device Flow
Support for OAuth 2.0 Device Authorization Grant flow for constrained devices.

### Verifiable Credentials
Issue and verify verifiable credentials using OpenID Connect for Verifiable Presentations.

## Error Handling

All API endpoints return consistent error responses with the following structure:

```json
{
  "resultCode": "A004001",
  "resultMessage": "Error description",
  "resultDescription": "Detailed error information"
}
```

Common error codes:

- **A004001**: Invalid request parameters
- **A004002**: Authentication required
- **A004003**: Insufficient permissions
- **A004004**: Resource not found
- **A004005**: Internal server error

## Rate Limiting

API calls are subject to rate limiting:

- **Service Management**: 1,000 requests/hour
- **Client Management**: 5,000 requests/hour
- **Authorization APIs**: 10,000 requests/hour
- **Token APIs**: 10,000 requests/hour
